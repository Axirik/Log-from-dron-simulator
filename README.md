# Log-from-dron-simulator

Небольшой учебный проект симуляции полёта дрона с передачей телеметрии по UDP, записью логов в CSV, визуализацией траектории и последующим анализом полёта.

В репозитории собраны **три этапа развития проекта**: базовая версия, версия с ПИД-регулятором и ручным/автоматическим управлением, а также финальная модульная версия с отдельным анализом логов и отладочными данными.

## Состав репозитория

### Базовая версия (корень репозитория)
- `simulator.py` — простой симулятор дрона с консольными командами, состояниями полёта и отправкой телеметрии по UDP
- `receiver.py` — приём телеметрии и запись в `flight_log.csv`
- `visualizer.py` — окно мониторинга с 3D-траекторией и графиками
- `sender.py` — упрощённый тестовый передатчик UDP
- `drone_visualizer.py` — дополнительный визуализатор
- `flight_log.csv` — пример сохранённого лога

### Версия 2: `drone_v2/`
- `simulator.py` — симулятор с ПИД-регулятором, режимами `AUTO`/`MANUAL`, ветром и расширенной отладкой
- `controller.py` — ручное управление с клавиатуры
- `receiver.py` — запись телеметрии в CSV
- `drone_visualizer.py` — улучшенный визуализатор с режимами LIVE/REPLAY и статистикой

### Версия 3: `drone_v3/`
- `Main.py` — основной файл запуска финальной версии
- `drone.py` — класс дрона с параметрами модели и состоянием
- `simulator.py` — модуль симуляции: физика, ПИД, состояния, ветер, орбита, логика управления
- `controller.py` — ручное управление с клавиатуры
- `receiver.py` — запись основной телеметрии в `flight_log.csv`
- `debug.py` — запись расширенных отладочных данных в `flight_debug.csv`
- `drone_visualizer.py` — финальный GUI-монитор с LIVE/REPLAY режимом
- `analyze.py` — постобработка логов и построение итогового графика `flight_analysis.png`
- `flight_log.csv`, `flight_debug.csv`, `flight_analysis.png` — примеры результатов работы

## О чём этот код

Проект моделирует движение квадрокоптера в упрощённой физической среде. Симулятор поддерживает:
- положение дрона по осям `X`, `Y`, `Z`
- изменение тяги, тангажа, крена и курса
- состояния полёта: `LANDED`, `TAKEOFF`, `HOVER`, `FLYING`, `LANDING`, `EMERGENCY`
- разряд батареи
- внешнее возмущение в виде ветра
- передачу телеметрии по UDP в реальном времени
- сохранение логов и их последующий анализ

Проект полезен как демонстрация:
- простой физической модели полёта
- телеметрии и UDP-обмена между процессами
- базового ПИД-управления
- визуализации и анализа данных полёта

## Эволюция версий

### Версия 1 — базовый прототип
Первая версия реализует:
- простую модель движения дрона
- ввод команд через консоль (`thrust`, `roll`, `pitch`, `yaw`)
- автоматический режим движения по кругу
- отправку телеметрии на визуализатор и логгер
- запись полётных данных в CSV
- отображение траектории и основных параметров

Это минимальный рабочий прототип, в котором уже есть симуляция, мониторинг и логирование.

### Версия 2 — управление и устойчивость
Во второй версии добавлены важные улучшения:
- ПИД-регулятор для движения к целевой точке
- режимы `AUTO` и `MANUAL`
- отдельный контроллер для ручного управления с клавиатуры
- задание целей по `X`, `Y`, `Z`
- моделирование ветра
- отдельный debug-поток с расширенной телеметрией
- улучшенный визуализатор с режимом просмотра логов (`REPLAY`)

По сравнению с первой версией проект стал не просто симулятором движения, а системой управления с возможностью анализа поведения регулятора.

### Версия 3 — финальная модульная реализация
В третьей версии проект стал более аккуратно структурирован:
- логика дрона вынесена в отдельный класс `Drone`
- логика симуляции вынесена в класс `Simulator`
- сохранено разделение на основную телеметрию и debug-данные
- визуализатор доработан для live-мониторинга и replay
- добавлен отдельный скрипт `analyze.py` для итогового анализа полёта
- появилась более чистая архитектура и удобство для дальнейшего расширения

### Что изменилось в новых версиях кратко
- **v1 → v2:** добавлены ПИД-регулятор, режимы `AUTO`/`MANUAL`, управление с клавиатуры, ветер и расширенная визуализация
- **v2 → v3:** код разделён на классы и модули, добавлены отдельные debug-логи и отдельный анализ полёта, структура проекта стала чище и удобнее

## Запуск финальной версии

Ниже приведён запуск именно для папки `drone_v3`.

### 1. Требования
Нужен Python 3.10+.

Библиотеки, которые используются в проекте:
```bash
pip install matplotlib pynput numpy
```

> `tkinter`, `socket`, `csv`, `threading`, `time` входят в стандартную библиотеку Python, но `tkinter` должен быть установлен в системе отдельно, если его нет.

### 2. Перейти в папку финальной версии
```bash
cd drone_v3
```

### 3. Запустить сервисы в разных терминалах

**Терминал 1 — запись основной телеметрии:**
```bash
python receiver.py
```

**Терминал 2 — запись debug-логов:**
```bash
python debug.py
```

**Терминал 3 — визуализатор:**
```bash
python drone_visualizer.py
```

**Терминал 4 — сам симулятор:**
```bash
python Main.py
```

После запуска можно управлять полётом через консоль команд в окне `Main.py`.

Примеры команд:
```text
 t=0.7   - увеличить тягу / взлёт
 tz=5    - задать целевую высоту 5 м
 tx=3    - сместить цель по X
 ty=2    - сместить цель по Y
 w=1     - включить ветер
 kp=2.0  - изменить коэффициент ПИД
 ki=0.5
 kd=1.0
 c=2     - включить полёт по кругу радиусом 2 м
```

### 4. Построить итоговый анализ после полёта
После накопления файлов `flight_log.csv` и `flight_debug.csv` можно построить итоговые графики:
```bash
python analyze.py
```

Результат будет сохранён в файл:
```text
flight_analysis.png
```

## Важное замечание
В текущем состоянии репозитория файл `drone_v3/Main.py` содержит поддержку режима `MANUAL`, но при этом в папке `drone_v3` **нет файла `controller.py`**, который он пытается запустить.

Поэтому:
- режим **AUTO** запускается и описан корректно
- для режима **MANUAL** потребуется запустить `controller.py` 

## Пример рабочего сценария
1. Запустить `receiver.py`
2. Запустить `debug.py`
3. Запустить `drone_visualizer.py`
4. Запустить `Main.py`
5. В консоли `Main.py` ввести:
```text
 tz=3
 tx=2
 ty=2
 c=2
```
6. После завершения полёта запустить:
```bash
python analyze.py
```
